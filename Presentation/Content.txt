Slide 3 — The Fiedler Value λ₂

Title:

Algebraic Connectivity λ₂ — One Number That Measures Swarm Coordination


Top — formula box (center, large):

Eigen decomposition of Laplacian L:
L·v = λ·v
Eigenvalues: 0 = λ₁ ≤ λ₂ ≤ λ₃ ≤ ... ≤ λₙ
λ₂ = Fiedler Value = Algebraic Connectivity

Swarm fully coordinated
❌ Dangerous

Panel 2 (orange border):

Sparse graph
Few links remaining
λ₂ = 0.21
Partially disrupted
⚠️ Degraded

Panel 3 (green border):

Two isolated clusters
Zero edges between them
λ₂ = 0.00
Swarm fragmented
✓ Mission complete


Bottom — Proposition 1 box (gold border):

Proposition 1 — Fiedler (1973):
λ₂(G) = 0 ⟺ Graph G is disconnected
λ₂(G) > 0 ⟺ Graph G is connected

When our system drives λ₂ to zero, swarm fragmentation is provably guaranteed.

Slide 10 — TD Error: The Foundation of Everything

Title:

Temporal Difference Error — How the Agent Measures Surprise


Top — intuition box (full width):

Core idea:
The Critic predicts expected future reward at state s_t
After one step, we see what actually happened
TD error = gap between prediction and reality


Center — formula derivation (step by step):
Step 1 — Critic prediction at time t:

V(s_t) = "From this state, I expect to earn X total future reward"

Step 2 — One step later, better estimate available:

G_t ≈ r_t + γ · V(s_{t+1})
r_t = actual reward earned this step
γ · V(s_{t+1}) = discounted value of new state
γ = 0.99 → future rewards worth slightly less than immediate

Step 3 — TD Error:

δ_t = r_t + γ · V(s_{t+1}) − V(s_t)
= (what actually happened) − (what Critic predicted)


Center — three interpretation boxes side by side:
Box 1 (green):

δ_t > 0
Reality was BETTER
than Critic predicted
Action was good →
increase its probability
Example:
Predicted 0.35
Got 0.15 + 0.99×0.37
δ = +0.017 ✓

Box 2 (grey):

δ_t = 0
Reality matched
Critic prediction exactly
Action was average →
no change needed
Critic is perfectly
calibrated for this state

Box 3 (red):

δ_t < 0
Reality was WORSE
than Critic predicted
Action was bad →
decrease its probability
Example:
Predicted 0.74
Got −0.05 + 0.99×0.68
δ = −0.057 ✗


Bottom strip (gold border):

δ_t is computed for EVERY timestep in the buffer after each rollout
It is the raw signal from which Advantage and GAE are both derived
Without δ_t there is no learning signal — it is the engine of all RL

Slide 11 — Advantage and GAE

Title:

From TD Error to GAE — Computing How Good Each Action Truly Was


Top — two column boxes:
Left box (problem):

Why not use δ_t directly as the learning signal?
δ_t is only ONE step of information
It fully trusts V(s_{t+1}) which is
estimated by the Critic
Early in training → Critic is wrong
→ δ_t is wrong → bad learning signal
We need to look further ahead
to get a more accurate signal

Right box (solution):

Advantage A_t:
A_t = "How much better was THIS action
compared to average for this state?"
A_t > 0 → action was above average → reinforce
A_t < 0 → action was below average → suppress
True advantage:
A_t = Q(s_t, a_t) − V(s_t)
But Q is unknown → we must estimate it


Center — GAE derivation box (teal border, large):

Generalised Advantage Estimation:
1-step estimate (TD):
A_t^(1) = δ_t = r_t + γV(s_{t+1}) − V(s_t)
2-step estimate:
A_t^(2) = δ_t + γδ_{t+1}
= r_t + γr_{t+1} + γ²V(s_{t+2}) − V(s_t)
n-step estimate:
A_t^(n) = δ_t + γδ_{t+1} + γ²δ_{t+2} + ... + γⁿ⁻¹δ_{t+n-1}
GAE — exponentially weighted sum of ALL n-step estimates:
A_t^GAE = (1−λ) · [A_t^(1) + λA_t^(2) + λ²A_t^(3) + ...]
Simplified recursive form (what code actually computes):
A_t^GAE = δ_t + (γλ) · A_{t+1}^GAE
Computed BACKWARDS from t=T to t=0


Center bottom — numerical example box (full width):

Example — your drone episode, last 4 steps:
δ₄₇ = −0.02,  δ₄₈ = +0.15,  δ₄₉ = +0.41,  δ₅₀ = 0 (terminal)
A₅₀ = 0
A₄₉ = δ₄₉ + (0.99×0.95)×0 = +0.41
A₄₈ = δ₄₈ + (0.94)×0.41  = 0.15 + 0.385 = +0.535
A₄₇ = δ₄₇ + (0.94)×0.535 = −0.02 + 0.503 = +0.483
Step 47 looks slightly bad in isolation (δ=−0.02)
But GAE shows it was actually good (+0.483)
because it set up the big reward at step 49
This is exactly why GAE is better than raw TD error

Slide 12 — PPO: Proximal Policy Optimization

Title:

PPO — Why Standard Policy Gradient Breaks and How PPO Fixes It


Top — vanilla policy gradient problem (full width, red border):

Standard Policy Gradient Update Rule:
θ ← θ + α · A_t · ∇_θ log π_θ(a_t | s_t)
In plain English:
"Move the weights in the direction that makes good actions more likely"
"The bigger the advantage A_t, the bigger the weight change"
Why this breaks:
Suppose A_t = 5.2 (very good action) and gradient is large.
Weight update is HUGE.
Policy jumps from "move toward centroid" to "always go to corner"
in a single update — completely destroying 50,000 steps of learning.
Next episode the policy is broken.
Rewards crash.
Takes thousands of steps to recover.
This is called a catastrophic policy update.
The problem is: we have NO control over how much the policy changes per update.


Center left — PPO core idea box (teal border):

PPO Key Insight:
Measure HOW MUCH the policy has changed from the old one.
If the change is too large → stop the update there.
Define the probability ratio:
r_t(θ) = π_θ(a_t | s_t) / π_θ_old(a_t | s_t)
r_t = 1.0 → policy unchanged for this action
r_t = 1.5 → new policy 50% MORE likely to take this action
r_t = 0.6 → new policy 40% LESS likely to take this action
PPO says: keep r_t inside [1−ε, 1+ε] = [0.8, 1.2]
No single update can change any action probability by more than 20%


Center right — clipped objective derivation (gold border):

Unclipped objective (naive):
L_UNCLIP = E_t[ r_t(θ) · A_t ]
Just scales advantage by how much policy changed.
No constraint → same catastrophic update problem.
Clipped objective:
L_CLIP = E_t[ min( r_t(θ)·A_t,  clip(r_t(θ), 1−ε, 1+ε)·A_t ) ]
The min() of two terms is the key:
Term 1: r_t·A_t → unconstrained objective
Term 2: clip(r_t, 0.8, 1.2)·A_t → hard-capped objective
min() always picks the MORE CONSERVATIVE of the two.
This creates a one-sided constraint:
→ Improvements are capped at 20%
→ The update cannot be exploited beyond the trust region




Bottom left — entropy box:

Entropy Bonus — keeping exploration alive:
H(π) = −Σ_a π(a|s) · log π(a|s)
High entropy → policy is spread across many actions → exploring
Low entropy → policy always picks same action → exploiting
Without entropy bonus:
Early in training policy collapses to one action
Never discovers better strategies
We ADD entropy to the loss:
L_entropy = −c₁ · H(π_θ)    c₁ = 0.01
Small penalty for being too certain too early


Bottom right — total loss box (green border):

Complete PPO Loss Function:
L_total = −L_CLIP + c₂·L_critic − c₁·L_entropy
Where:
L_CLIP   = clipped policy objective (maximize)
L_critic = MSE( V(s_t) , G_t )² (minimize)
L_entropy = entropy bonus (maximize)
c₁ = 0.01 → entropy weight
c₂ = 0.5  → critic loss weight
Negative sign on L_CLIP because
PyTorch minimizes loss →
we negate to turn maximization into minimization

Slide 13— The Complete PPO Training Loop

Title:

PPO Training — One Complete Update Cycle


Center — clean numbered flowchart (6 steps):
Step 1 (blue box):

Collect Experience
All 6 jammers run for 2048 steps using current policy
Store at every step: state, action, reward, log probability, critic value

↓
Step 2 (orange box):

Compute GAE Advantages
Walk backwards through buffer
A_t = δ_t + γλ · A_{t+1}
Every step now has an advantage estimate

↓
Step 3 (teal box):

Normalize Advantages
A_t ← (A_t − mean) / std
All advantages on same scale → stable gradients

↓
Step 4 (purple box):

10 Epochs of Mini-batch Updates
Shuffle buffer → split into batches of 256
For each batch:
→ Compute ratio r_t = π_new / π_old
→ Compute clipped PPO loss
→ Compute Critic MSE loss
→ Backpropagate → update weights

↓
Step 5 (red box):

Gradient Clip + Adam Update
Clip gradient norm to 0.5
Adam adjusts each weight individually
W_new = W_old − lr · gradient

↓
Step 6 (green box):

Reset and Repeat
π_old ← π_new
Clear buffer
Back to Step 1
